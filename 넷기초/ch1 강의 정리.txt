-데이터 통신의 정의-
>데이터가 A -> B 지점으로 이동하는 것
>최소한 하나 이상의 전송 매체가 필요(무선/유선)
>전송매체에 얹을 수 있는 형태로 포멧이 되어야 함
>HW, SW, 서비스가 포함이 됨
>포멧된 데이터 = 인코딩된 데이터 -> 전송 특화 


- 비트, 바이트, 데이터 인코딩 -
>비트: 가장 작은 단위
>바이트: 비트 * 8
>데이터 인코딩: 비트 또는 바이트를 이용 -> 디지털 방식으로 포멧팅

>EBCDIC: 옛날 방식
>ASCII: 1바이트에 할당
>Unicode: 2바이트에 할당

>인코딩 <-> 디코딩
>인코딩: 정보를 묶고 압축
>디코딩: 정보를 풀고 봄


- 디지털/아날로그 데이터 -
>아날로그 데이터: 
>>연속적인 모든 값을 가질 수 있다, 표현량 많음, 신호 보강 어려움 -> 값이 다양하기 때문

>디지털 데이터: 
>>0 또는 1의 값만을 가질 수 있다, 표현량 제한적, 신호 보강 쉬움 -> 0과 1밖에 없기 때문

>아날로그 전송 방식:
>>ex) 사람 목소리

>디지털 전송 방식:
>>ex)컴퓨터 데이터
>>모뎀: Modulator + Demodulator = Modem

>병렬 전송 방식: 1바이트의 데이터를 한 번에 전송, 과거에 주로 사용, ex) 구형 프린터 케이블
>직렬 전송 방식: 1바이트의 데이터를 일렬로 전송, 현재 주로 사용, ex) USB

>비동기 전송 방식: 받는 사람의 동의 여부에 관계 없이 전송, ex) SNS
>동기 전송 방식: 받는 사람의 동의가 있을 때 전송, ex) 전화 통화

>Simplex(단이중): 한 방향으로만 통신 가능
>Half-Duplex(반이중): 라인이 1개, 양방향 통신 가능, 한 번에 한쪽 방향으로만 통신 가능 ,ex) 무전기
>Full-Duplex(전이중): 항상 양방향 통신 가능

- 데이터 통신 표준 -
>받아들여진 모델 / 패턴
>데이터에 광범위하게 사용
>기본 수준의 디바이스 간 호환성 및 상호 운용성 제공

>ASNI: ASCII 제정
>IEEE: 전기, 전자, 컴퓨터 관련 엔지니어들이 모여 표준 논의
>ITU: 장비 제조업체 포함
>ISO: 전 세계 표준 제정


- 데이터 통신 모델 -

계층 구조로 되어있음

OSI: 이론적인 모델
>1970후반 정의
>7개의 계층
>각각의 계층은 프로토콜 규약으로 구성

-OSI::7개 계층
>7 - 운용 계층
>6 - 표현 계층
>5- 세션 계층
>4 - 전송 계층
>3 - 네트워크 계층
>2 - 데이터 링크 계층
>1 - 물리 계층    

-OSI::물리 계층
>OSI 모델의 계층 1이라고도 함
>장치 간 비트의 물리적 연결 및 전송을 관리하는 프로토콜 정의
>디지털 또는 아날로그 등의 신호 방식 정의
>비동기식, 동기식, 심플렉스, 반이중 또는 전이중과 같은 전송 특성 지정
>10Mbps, 100Mbps, 1000Mbps 등의 데이터 속도를 정의

-OSI::데이터 링크 계층
>물리 계층의 데이터를 받아 상위 계층으로 전달
>비트 -> 프레임 전환 (data in)
>프레임 -> 비트 전환 (data out)
>노드 주소 정의
>에러 검출과 수정
>데이터 비트 접근 방법 정의, 예) NetWork Adapter

-OSI::네트워크 계층
>논리 네트워크와 노드 주소 지정 정의
>패킷 생성과 패킷 순서 지정
>데이터 링크 계층의 데이터를 받아 전송 계층으로 전달
>네트워크 간 경로 검색 및 최적 경로 결정 기능을 제공

-OSI::전송 계층
>상위 계층에서 메시지를 수신하여 해당 메시지를 더 작은 청크로 분할
>연결 중심의 데이터 서비스 제공
>end-to-end 흐름 제어 제공
>서비스 주소 또는 포트 번호 식별

-OSI::세션 계층
>두 장치 간의 통신 설정, 유지, 동기화 및 종료 담당

-OSI::표현 계층
>인코딩 ASCII, EBCDIC 또는 유니코드 같은 데이터 변환 서비스를 제공(인코딩)
>데이터 전송 내에서 end-to-end 암호화 서비스 제공

-OSI::운용 계층
>사용자 SW를 지원하는 서비스인 파일, 인쇄, 이메일 서비스를 제공
>원격 액세스 서비스가 있음
>협업 컴퓨팅 서비스 및 서비스와 광고 메커니즘이 해당



- 데이터 캡슐화 -
>계층화된 아키텍처에서 각 계층에 대한 데이터 비트 세트에 헤더로 알려진 
추가 프로토콜 정보 세트를 추가하는 프로세스

>각 계층의 프로토콜은 데이터를 교환하는 둘 이상의 장치에서 실행되는 
유사한 프로세스, 서비스 또는 기능 간에 데이터 통신이 어떻게 이루어져야 하는지를 
설명하는 프레임워크 제공

>각 계층을 설명하는 규칙에 따라 기능하는 프로토콜은 통신 장치 간의 데이터 교환을 용이하게 함


TCP/IP: 실제적인 모델
>1970초반 정의
>디바이스 간 통신 기능을 정의하기 위해 계층화된 구조 사용
>공식 표준이 아님
>5개 또는 4개의 계층

-TCP/IP::계층
>4 - 프로세스/운용 
>3 - host-to-host
>2 - 인터넷
>1 - 네트워크 접근


== 2장 ==

PAN: Personal Area Networks
LAN: Local Area Networks
MAN: Metropolitan Area Networks
WAN: Wide Area Networks

- LAN -
>전용 HW, SW 활용
>>서버 및 워크스테이션 기능을 하는 컴퓨터
>>서비스를 제공하는 OS
>>LAN에 연결되는 네트워크 인터페이스 카드
>>허브, 브릿지, 스위치, 라우터, 다른 연결 기기

- 서버 -
>up and running <-> server down
>LAN에 연결하고 네트워크의 클라이언트에 공유 LAN 서비스를 제공하기 위한 
네트워크 운영 체제 소프트웨어가 설치된 컴퓨터

- 랜 서비스 -
>LAN의 속성 정의

- 클라이언트 -
>서버의 네트워크 운영 체제에서 제공하는 LAN 서비스에 사용자를 연결하는 컴퓨터


- LAN 구성 -
>메인프레임 및 터미널
>>메인프레임에서 컴퓨팅 기능이 수행되며 터미널은 데이터 입력 및 데이터 보기 지원
>>>메인프레임에서 처리가 이루어지기 때문에 터미널은 "멍청한" 것으로 간주
>>>메인프레임은 주어진 자본 지출 수준에 대해 고정된 양의 컴퓨팅 성능을 제공
>>>처리 성능 업그레이드에 비용이 많이 듬
>>>애플리케이션 개발에 비용이 많이 듬
>>>메인프레임 기술은 오늘날의 LAN에서 데이터 스토리지 기능을 위한 
스토리지 영역 네트워킹으로 채택

- Peer-to-Peer LANs -
>전용 메인프레임 또는 서버에서 중앙 집중식 컴퓨팅 성능을 
필요로 하지 않는 조직의 비즈니스 요구를 충족
>>P2P LAN을 사용하면 LAN의 각 컴퓨터는 다른 모든 컴퓨터의 피어 역할을 함
>>각 컴퓨터는 LAN 상의 모든 다른 컴퓨터에 서비스를 제공하고 서비스를 요청할 수 있음
>>피어 컴퓨터는 일반적으로 동일한 워크그룹에 속하도록 구성

- Clinent-dominant LANs -
>1980년대에 개별 PC용으로 개발된 단일 사용자 애플리케이션에서 발전
>LAN 서버의 데이터 스토리지는 중앙 집중식 파일 액세스를 제공
>애플리케이션 처리 및 데이터 조작이 클라이언트에서 발생
>사용자의 PC 간 "스니커넷"에 대한 의존도 감소

- Client/Server LANs -
>일부 처리는 클라이언트에서 수행되고 일부는 서버에서 수행
>서버는 데이터 액세스 및 저장을 처리
>데이터 조작을 위해 필요한 정보만 클라이언트 컴퓨터에 복사
>LAN에 대한 이점은 네트워크 미디어에 대한 수요 감소와 전반적인 네트워크 성능 향상

- 분산 처리 LANs -
>데이터 액세스 및 저장 컴포넌트들은 애플리케이션의 데이터 프로세싱 컴포넌트로부터 분리
>응용 프로그램 처리는 여러 컴퓨터에 걸쳐 공유
>애플리케이션의 상이한 컴포넌트들은 하나 이상의 서버들에 설치될 수 있음
>애플리케이션에 내장된 로직을 통해 모든 컴포넌트 모듈이 통신할 수 있음
>애플리케이션의 클라이언트 구성 요소는 실제 처리가 어디에서 이루어지는지를 신경 쓰지 않음
>이러한 유형의 LAN은 증가하는 처리 수요에 잘 확장
>로드 공유 및 이중화로 향상된 성능을 제공

- LAN에 컴퓨터 연결 -
>클라이언트 컴퓨터에는 워크스테이션 OS와 NOS 클라이언트 필요
>서버는 NOS 설치 및 구성 필요
>네트워크 관리자는 클라이언트 소프트웨어 및 하드웨어, 
서버 하드웨어 및 소프트웨어, 프로토콜, NIC, 미디어 및 연결 장치 구성


-- 전송 매체 --

>동축 케이블 -
>>일반적으로 "coax"
>>특수 절연 소재로 분리된 두 개의 도체로 구성
>>한 도체는 신호를 전달하고 다른 도체는 접지 및 차폐 역할
>>초기 이더넷 LAN에 많이 사용하였음

>Thicknet
>>두꺼운 이더넷 케이블로 알려져 있고 500미터 거리에서 
최대 10Mbps의 데이터 전송 속도를 제공하고 LAN 세그먼트에서 100대의 컴퓨터 연결을 지원

>Thinnet
>>얇은 이더넷 케이블로 알려져 있고 Thicknet의 대체제로 1985년에 도입
>>Thicknet보다 저렴함
>>185미터 내에서만 최대 10Mbps의 전송속도 제공
>>최대 30개의 컴퓨터 연결 지원

>Thinnet Connector
>>BNC 커넥터를 사용하여 케이블을 케이블에 연결하고 
NIC를 케이블에 연결하며 연결 장치를 케이블에 연결

>Twisted Pair Media
>>LAN 설치에서 매우 인기가 많음
>>여러 쌍의 꼬인 구리 와이어로 구성
>>차폐되지 않은 종류과 차폐된 종류 
>>>UTP(Unshielded Twisted Pair) / STP(Shielded Twisted Pair)

>UTP Connectors
>>UTP 8핀 커넥터로 종단처리
>>일반적으로 RJ-45 커넥터로 불림

>Shilded Twisted Pair
>>UTP와 동일한 연결 이점을 제공하지만 
STP는 EMI(Electromagnetic Interference)로부터 데이터 전송을 보호하기 위해 
두 단계의 차폐 소재를 추가

>STP Connectors
>>UTP 커넥터와 매우 유사하지만, STP 커넥터는 
STP 케이블의 차폐가 연결되는 접지 케이스를 제공

>광케이블
>>빛의 파동을 전송하기 위해 유리 또는 플라스틱 섬유 사용
>>높은 데이터 전송률과 대용량 데이터 전송이 필요한 곳에서 사용
>>일반적으로 송신 장치와 수신 장치 사이의 두 가지의 광케이블 존재 - 송신용 / 수신용
>>EMI의 영향을 받지 않음
>>구리 케이블보다 더 먼 거리와 더 높은 전송 속도 지원

>무선 매체
>>전자적 또는 광학적 신호를 보내는 컨덕터가 필요 없음
>>무선 주파수, 적외선 통신 및 마이크로 주파수 포함

>>무선 주파수
>>>각 LAN 장치에는 송수신기와 안테나가 있음
>>>무선랜 무선 주파수는 무선국에 간섭하지 않음
>>>무선 주파수는 KCC(방송통신위원회)에 의해 할당
>>>무선 장치는 LAN에 연결하기 위해 액세스 포인트 사용
>>>액세스 포인트 -> 셀(Cell)

>>적외선 통신
>>>데이터 전송을 위해 가벼운 주파수를 사용
>>>일반적으로 매우 짧은 거리로 제한
>>>다양한 유형의 간섭에 취약
>>>포인트 투 포인트 또는 방송 전송에 사용할 수 있음

>>마이크로 주파수
>>>데이터 전송을 위해 매우 높은 주파수의 전파 사용
>>>LAN 애플리케이션들에서 사용될 때, 일반적으로 빌딩들 사이의 전송들을 위해 설치
>>>지상파 마이크로파는 포물선 안테나를 사용하여 데이터를 전송

>>Wireless Connectors
>>>무선 데이터 전송은 동축, 트위스트 페어 및 광섬유 케이블에 사용되는 
커넥터와 유사한 커넥터를 사용하지 않음
>>>대신에 무선 디바이스들은 안테나들과 트랜시버들을 사용하여 
무선 디바이스들 사이에 접속을 형성
