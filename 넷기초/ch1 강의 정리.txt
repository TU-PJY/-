-데이터 통신의 정의-
>데이터가 A -> B 지점으로 이동하는 것
>최소한 하나 이상의 전송 매체가 필요(무선/유선)
>전송매체에 얹을 수 있는 형태로 포멧이 되어야 함
>HW, SW, 서비스가 포함이 됨
>포멧된 데이터 = 인코딩된 데이터 -> 전송 특화 


- 비트, 바이트, 데이터 인코딩 -
>비트: 가장 작은 단위
>바이트: 비트 * 8
>데이터 인코딩: 비트 또는 바이트를 이용 -> 디지털 방식으로 포멧팅

>EBCDIC: 옛날 방식
>ASCII: 1바이트에 할당
>Unicode: 2바이트에 할당

>인코딩 <-> 디코딩
>인코딩: 정보를 묶고 압축
>디코딩: 정보를 풀고 봄


- 디지털/아날로그 데이터 -
>아날로그 데이터: 
>>연속적인 모든 값을 가질 수 있다, 표현량 많음, 신호 보강 어려움 -> 값이 다양하기 때문

>디지털 데이터: 
>>0 또는 1의 값만을 가질 수 있다, 표현량 제한적, 신호 보강 쉬움 -> 0과 1밖에 없기 때문

>아날로그 전송 방식:
>>ex) 사람 목소리

>디지털 전송 방식:
>>ex)컴퓨터 데이터
>>모뎀: Modulator + Demodulator = Modem

>병렬 전송 방식: 1바이트의 데이터를 한 번에 전송, 과거에 주로 사용, ex) 구형 프린터 케이블
>직렬 전송 방식: 1바이트의 데이터를 일렬로 전송, 현재 주로 사용, ex) USB

>비동기 전송 방식: 받는 사람의 동의 여부에 관계 없이 전송, ex) SNS
>동기 전송 방식: 받는 사람의 동의가 있을 때 전송, ex) 전화 통화

>Simplex(단이중): 한 방향으로만 통신 가능
>Half-Duplex(반이중): 라인이 1개, 양방향 통신 가능, 한 번에 한쪽 방향으로만 통신 가능 ,ex) 무전기
>Full-Duplex(전이중): 항상 양방향 통신 가능


2024.03.14

- 데이터 통신 표준 -
>받아들여진 모델 / 패턴
>데이터에 광범위하게 사용
>기본 수준의 디바이스 간 호환성 및 상호 운용성 제공

>ANSI: 국가 표준을 추구하는 회원사를 대표
>IEEE: 전기, 컴퓨터 및 제어 표준의 개발
>ITU: 수많은 데이터 통신 표준의 표준화를 지원
>ISO: 데이터 통신 기술에 대한 표준을 개발하고 발표


- 데이터 통신 모델 -

계층 구조로 되어있음

OSI: 이론적인 모델
>1970후반 정의
>7개의 계층
>각각의 계층은 프로토콜 규약으로 구성

-OSI::7개 계층
>7 - 운용 계층
>6 - 표현 계층
>5- 세션 계층
>4 - 전송 계층
>3 - 네트워크 계층
>2 - 데이터 링크 계층
>1 - 물리 계층    

-OSI::물리 계층
>OSI 모델의 계층 1이라고도 함
>장치 간 비트의 물리적 연결 및 전송을 관리하는 프로토콜 정의
>디지털 또는 아날로그 등의 신호 방식 정의
>비동기식, 동기식, 단이중, 반이중 또는 전이중과 같은 전송 특성 지정
>10Mbps, 100Mbps, 1000Mbps 등의 데이터 속도를 정의

-OSI::데이터 링크 계층
>물리 계층의 데이터를 받아 상위 계층으로 전달
>비트 -> 프레임 전환 (data in)
//>프레임 -> 비트 전환 (data out)
>노드 주소 정의
>에러 검출과 수정
>데이터 비트 접근 방법 정의, 예) NetWork Adapter

-OSI::네트워크 계층
>논리 네트워크와 노드 주소 지정 정의
>패킷 생성과 패킷 순서 지정
>데이터 링크 계층의 데이터를 받아 전송 계층으로 전달
>네트워크 간 경로 검색 및 최적 경로 결정 기능을 제공

-OSI::전송 계층
>상위 계층에서 메시지를 수신하여 해당 메시지를 더 작은 청크로 분할
>연결 중심의 데이터 서비스 제공
>end-to-end 흐름 제어 제공
>서비스 주소 또는 포트 번호 식별

-OSI::세션 계층
>두 장치 간의 통신 설정, 유지, 동기화 및 종료 담당

-OSI::표현 계층
>인코딩 ASCII, EBCDIC 또는 유니코드 같은 데이터 변환 서비스를 제공(인코딩)
>데이터 전송 내에서 end-to-end 암호화 서비스 제공

-OSI::운용 계층
>사용자 SW를 지원하는 서비스인 파일, 인쇄, 이메일 서비스를 제공
>원격 액세스 서비스가 있음
>협업 컴퓨팅 서비스 및 서비스와 광고 메커니즘이 해당



- 데이터 캡슐화 -
>계층화된 아키텍처에서 각 계층에 대한 데이터 비트 세트에 헤더로 알려진 
추가 프로토콜 정보 세트를 추가하는 프로세스

>각 계층의 프로토콜은 데이터를 교환하는 둘 이상의 장치에서 실행되는 
유사한 프로세스, 서비스 또는 기능 간에 데이터 통신이 어떻게 이루어져야 하는지를 
설명하는 프레임워크 제공

>각 계층을 설명하는 규칙에 따라 기능하는 프로토콜은 통신 장치 간의 데이터 교환을 용이하게 함

TCP/IP: 실제적인 모델
>1970초반 정의
>디바이스 간 통신 기능을 정의하기 위해 계층화된 구조 사용
>공식 표준이 아님
>5개 또는 4개의 계층

-TCP/IP::계층
>4 - 프로세스/운용 
>3 - host-to-host
>2 - 인터넷
>1 - 네트워크 접근
